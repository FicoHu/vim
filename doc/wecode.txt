					*Wecode* *wecode* *Wecode-doc*
Wecode is a self-hosted AI coding assistant that can suggest multi-line code or
full functions in real-time. For more information, please check out our
{Website}{1} and {Github}{2}. If you encounter any problem or have any
suggestion, please {open_an_issue}{3} or join our {Slack_community}{4} for
support.
 

					*Wecode-compatibility* *Wecode-NeoVim*
Compatibility~
This plugin is compatible with VIM 9.0+ with `+job` and `+textprop` features
enabled, or NeoVIM 0.6.0+.

					*Wecode-commands*
Commands~
                                        *:Wecode*
:Wecode			Same as |:Wecode-status|.
					*:Wecode-status*
:Wecode  status		Check the status of Wecode. Report error message if any 
			issue exists.
					*:Wecode-version*
:Wecode  version		Print the version of Wecode plugin.
					*:Wecode-help*
:Wecode  help [subject]  Search for help information in this document using
			VIM command `:help`.

					*Wecode-options*
Options~
					*g:wecode_node_binary* *Wecode-node*
g:wecode_node_binary	Wecode plugin will try to find the Node.js binary in 
			your `PATH` environment variable. If you have installed 
			Node.js in a non-standard location, or you are using a 
			version manager such as nvm, you can set the Node.js 
			binary path here.
>
	let g:wecode_node_binary = '/path/to/node'
<
					*g:wecode_trigger_mode* *Wecode-trigger*
g:wecode_trigger_mode	Completion trigger mode.
			- 'auto': Trigger completion automatically when you
				stop typing.
			- 'manual': Only trigger completion when you press
				`<C-\>`.
			Default value is 'auto'.
>
	let g:wecode_trigger_mode = 'auto'
	let g:wecode_trigger_mode = 'manual'
<
					*g:wecode_filetype_dict* *Wecode-filetype*
g:wecode_filetype_dict	This option is a dictionary that map from the Vim
			`:filetype` to {VSCode_Language_Identifier}{1}. Not
			listed filetype will be used as language identifier
			directly.
			The following mappings are provided by default, you can
			add your own mappings to override them.
  {1}  https://code.visualstudio.com/docs/languages/identifiers
>
	let g:wecode_filetype_dict = #{
		\ bash: "shellscript",
		\ sh: "shellscript",
		\ cs: "csharp",
		\ objc: "objective-c",
		\ objcpp: "objective-cpp",
		\ make: "makefile",
		\ cuda: "cuda-cpp",
		\ text: "plaintext",
		\ }
<
					*Wecode-keybindings* *Wecode-map*
Keybindings~

<Tab>			Accept the current completion, fallback to normal
			`<Tab>` if no completion is shown.

<C-\>			Trigger completion if not shown. Dismiss the current 
			completion if shown.
